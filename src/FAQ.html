FAQ


Q: What input file types are supported?

A: Currently supported input file types are XLS, XLSX and CSV.


Q: In what format should I input the data?

A: The file should contain four columns and no header. In CSV, columns should be separated by comma.

All entries should be numeric with decimals separated by dot. The columns should present: 1) the distance from Bregma, 2) the distance from the medial reference point to medial edge of the lesion (M1), 3) from the medial edge of the lesion to the lateral edge of the lesion (M2), and 4) the distance from the lateral edge of the lesion to the rhinal fissure (M3).


Q: Where can I find example inputs

A: Synthetic examples are provided at the project github page for a timeseries of measurements from days 7, 14, and 21
